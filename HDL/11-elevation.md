# HDL 11 — Elevation & Materials

## Purpose

Shadows, blur, glass effects, and material hierarchy for surfaces.

## Status

**FUTURE**

## Reasoning

Home Experience v2 intentionally avoids card elevation and shadows — the flowing layout uses whitespace and typography for hierarchy instead. Elevation tokens should be designed when screens genuinely need depth (modals, sheets, overlays).

## Validation notes

Not yet designed. No screens currently require elevation tokens.

## Implementation notes

No token file yet. Do not define shadow or blur values until validated in context.
